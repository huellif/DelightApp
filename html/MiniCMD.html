MiniCMD commands

abort

Effectively function: abort MiniCMD continue to perform the abort command statement (only the current file command set)
Form: abort

att

Function: Set the files or file attributes
Form: att [parameter list] [destination path]

List of available parameters:

+S Add System Properties
+H plus hidden attribute
+R plus the read-only attribute (corresponding to + be changed - that means removing attributes)
/S including subdirectories

Example: remove c:\ data under the read-only attribute of all files

att -r /s c:\data\*

cmd

Functions: command set dynamically load the specified file
Form: cmd [file path]

Such as

if XXX
cmd ddd.txt
endif

At this point, only executed only if the establishment cmd, rather than like the previous version into established loaded before execution

Example: Loading e:\other\my.txt command

cmd e:\other\my.txt

In addition, you can not load another file the original! ! !

For example, in C:\A.txt cmd in C:\B.txt can not write in the B cmd in C:\A.txt! ! !

cp or copy

Function: Copy a file or folder
Form: cp [parameter list] [source path] [destination path]

List of available parameters:

/S including subdirectories, copy directories including subdirectories, it is only used when copying files
/O -o overwrite existing files without this parameter is not overwritten

Support at the same time set the new file attributes, the same parameter att

Example: Copy z:\ data's. Aac aac files to e plate folder and remove the read-only attribute

cp /s -r z:\data\*.aac e:\aac \

delete

Function: Delete Files

Example:

delete e:\test\*.txt

log

Function: Output his party Log
Form: log [output]

mv

Function: move the file or folder
Form of parameters with cp

Example: e:\music files\lyrics (. Lrc) move to the e:\lyrics

md or mkdir

Features: create directory
Form: md [directory]

Example: create e plate test \ subfolder directory

rm or rmdir

Function: delete a file or folder
The form: rm [parameter list] [destination path]

List of available parameters:

/S including subdirectories, should only be used to delete files

Example: delete c: \ other all sis installation package

rm /s c:\other\*.sis?

rn or rename

Functions: rename a file or folder
The form: of rn [source path] [destination path]

Example: rename e:\ the lyrics lyrics file suffix (.Lrc).Txt

rn e:\lyrics\*.lrc e: \lyrics\*.txt

run

Function: calling application
Form: run [UID or program name]

List of available parameters:

-C, in theory only applied to the running daemon

The name of the program or without the path

Example: run UC

run UcWeb60Signed.exe

kill

Function: the end of the process
Forms: kill [UID or process name

Where UID must 0x (0X) at the beginning of the hexadecimal

Example: the end of MIFEditor

sleep

Functions: pause the program
Form: sleep [milliseconds] to

Example: Pause 3 seconds

find

find [parameters] [path | file] [operation]

: Find files and corresponding

The current operation only supports rm and log the parameters only / s, when added at the same time search subdirectories

[Path to file] format: You can have more than one path separated by a semicolon (;)

The | separated between the path with the file such as Find txt files under the C drive with the E drive and delete

Should be noted that, the path must have \ (such as the above C: \) path with Less, operating only lowercase

Rm operating delete the directory does not support (assuming you find the directory)

Jump command

abort_if_exists *, abort_if_not_exists

The following command to interrupt the currently loaded file when the specified file exists (does not exist)

Example:

abort_if_exists c:\data\test.bin

skip_if_exists, skip_if_not_exists

Specified file (not) exist, skip to the next command

back_if_exists, back_if_not_exists

Specified file (not) exist to return to the previous one command

restart_if_exists, restart_if_not_exists

Specified file (not) exist re-execute all commands (that is to start from scratch again)

(Please be careful to use the above two sets of commands (highlighted in green), or it may cause an infinite loop eventually cause MiniCMD crash was not working properly)

(And limit these two sets of commands can not be the first, or simply ignored)
